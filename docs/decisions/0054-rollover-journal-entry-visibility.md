# 0054 - Rollover Journal Entry Visibility

## Status
Accepted

## Context
Academic year rollover execution creates closing and opening journal entries through backend services. Operators and auditors need a direct way to see which journal entries were generated by a rollover without exposing new posting or reversal controls inside the rollover workflow.

## Decision
The rollover detail page now shows a read-only "Generated Journal Entries" section. It lists journal entries returned by the accounting API where:

- `source_app=academic`
- `source_model=AcademicYearRollover`
- `source_object_id` matches the rollover id
- `source_type=system`

The accounting journal list endpoint supports exact read-only source filters for those fields while preserving organization and branch scoping. Journal responses also include read-only debit and credit totals so the UI can display totals from backend journal data.

## Behavior
The frontend does not calculate closing entries, opening entries, or ledger effects. It only fetches journal metadata and totals from the backend and links to the journal detail page. Rollover-generated entries are displayed as read-only rows with entry number, type, date, status, description, debit total, and credit total.

## Limitations
The UI classifies entries as opening or closing from the backend `source_number` convention. If future rollover services introduce more source numbers, they will appear as generic rollover entries until the display mapping is extended.
